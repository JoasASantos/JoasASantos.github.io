## Hacks & Techniques

### 1. Email Header Analysis for Tracing Origin

Email header analysis is a fundamental forensic technique for tracing the origin of an email, identifying spoofing attempts, and understanding the email's routing path.

#### Email Header Structure

Emails contain headers that record the path the message traveled from sender to recipient. Headers are added in reverse chronological order (newest at the top).

#### Key Header Fields for Forensic Analysis

| Header Field | Description | Forensic Value |
|---|---|---|
| **Received** | Each mail server adds a Received header | Traces the email's path from origin to destination |
| **X-Originating-IP** | IP address of the sending client | Identifies the sender's actual IP address |
| **Message-ID** | Unique identifier generated by the sending server | Can reveal the sending mail server |
| **Return-Path** | Where bounced emails are sent | May differ from the From address (spoofing indicator) |
| **DKIM-Signature** | DomainKeys Identified Mail signature | Verifies the email hasn't been tampered with |
| **Authentication-Results** | SPF, DKIM, and DMARC results | Shows if the email passed authentication checks |
| **X-Mailer** | Email client used to compose the message | Can identify the sender's software |

#### Reading Received Headers

```
Received: from mail-server3.example.com (10.0.0.3)
    by recipient-mx.corp.com (10.1.0.1) with ESMTPS
    id abc123; Mon, 6 Oct 2019 13:00:05 -0400     <-- Final hop (newest)

Received: from mail-server2.relay.net (172.16.0.50)
    by mail-server3.example.com (10.0.0.3) with ESMTP
    id def456; Mon, 6 Oct 2019 12:59:58 -0400     <-- Middle relay

Received: from sender-machine.local (203.0.113.42)
    by mail-server2.relay.net (172.16.0.50) with SMTP
    id ghi789; Mon, 6 Oct 2019 12:59:50 -0400     <-- First hop (sender's IP: 203.0.113.42)
```

#### Spoofing Detection

- **SPF (Sender Policy Framework):** DNS record that specifies which IP addresses are authorized to send email for a domain.
- **DKIM (DomainKeys Identified Mail):** Cryptographic signature that verifies the email body and key headers haven't been altered.
- **DMARC (Domain-based Message Authentication, Reporting, and Conformance):** Policy that tells receiving servers what to do when SPF or DKIM checks fail.

### 2. Whiterose's ICS/SCADA Infrastructure at Congo Facility

The episode reveals more about Whiterose's mysterious project, which involves industrial control systems at a facility in the Congo.

#### ICS/SCADA Overview

- **ICS (Industrial Control Systems):** The general term for control systems used in industrial processes, including manufacturing, power generation, water treatment, and mining.
- **SCADA (Supervisory Control and Data Acquisition):** A subcategory of ICS that provides centralized monitoring and control of distributed processes.
- **PLC (Programmable Logic Controller):** The industrial computer that directly controls physical processes (motors, valves, actuators).
- **HMI (Human-Machine Interface):** The graphical interface operators use to monitor and control the industrial process.
- **RTU (Remote Terminal Unit):** Field device that interfaces between physical equipment and the SCADA system.

#### ICS/SCADA Network Architecture (Purdue Model)

```
Level 5: Enterprise Network (Internet, Corporate IT)
         |
    --- DMZ / Firewall ---
         |
Level 4: Site Business Planning (ERP, Email, Business Apps)
         |
    --- Firewall ---
         |
Level 3: Site Operations (Historian, Engineering Workstations)
         |
Level 2: Area Supervisory Control (HMI, SCADA Servers)
         |
Level 1: Basic Control (PLCs, RTUs, Controllers)
         |
Level 0: Physical Process (Sensors, Actuators, Valves, Motors)
```

#### Common ICS Protocols

| Protocol | Port | Description |
|---|---|---|
| **Modbus TCP** | 502 | Most common ICS protocol; no authentication |
| **DNP3** | 20000 | Used in utility SCADA systems |
| **EtherNet/IP** | 44818 | Used in manufacturing automation |
| **OPC UA** | 4840 | Modern ICS data exchange protocol |
| **S7comm** | 102 | Siemens S7 PLC communication |
| **BACnet** | 47808 | Building automation and HVAC control |
| **PROFINET** | Various | Real-time industrial Ethernet (Siemens) |

#### ICS Vulnerabilities

- **Lack of Authentication:** Many ICS protocols (especially Modbus) have no built-in authentication.
- **Unencrypted Communications:** Most ICS protocols transmit data in plaintext.
- **Legacy Systems:** Many ICS environments run decades-old software that cannot be patched.
- **Flat Networks:** Inadequate network segmentation between IT and OT (Operational Technology) networks.
- **Default Credentials:** PLCs, HMIs, and engineering workstations often retain factory default passwords.

### 3. Encrypted File Systems (VeraCrypt, LUKS)

Characters use encrypted file systems to protect sensitive data, with specific emphasis on plausible deniability through hidden volumes.

#### VeraCrypt

VeraCrypt is the successor to TrueCrypt, providing on-the-fly disk encryption:

- **Full Disk Encryption (FDE):** Encrypts the entire operating system partition including the OS itself.
- **Encrypted Containers:** Creates encrypted virtual disks stored as files that can be mounted as real drives.
- **Partition Encryption:** Encrypts entire non-system partitions.
- **Encryption Algorithms:** AES-256, Serpent, Twofish, Camellia, and cascaded combinations (e.g., AES-Twofish-Serpent).
- **Hash Algorithms:** SHA-512, SHA-256, Whirlpool, Streebog for key derivation.
- **PIM (Personal Iterations Multiplier):** Customizable iteration count for key derivation, adding another layer against brute-force.

#### LUKS (Linux Unified Key Setup)

LUKS is the standard for Linux hard disk encryption:

- **Header Format:** Stores encryption metadata (cipher, key slots, salt) in a standardized header.
- **Multiple Key Slots:** Up to 8 different passphrases can unlock the same volume.
- **Anti-Forensic Information Splitter:** Makes key material recovery from discarded key slots computationally infeasible.
- **Cipher Modes:** AES-XTS (default in LUKS2), Serpent, Twofish.

### 4. Hidden Encrypted Volumes for Plausible Deniability

The concept of hidden encrypted volumes provides plausible deniability: the ability to convincingly deny that a hidden volume exists even under coercion.

#### VeraCrypt Hidden Volume Architecture

```
+--------------------------------------------------+
|              Outer Volume (Decoy)                 |
|  Password: "decoy_password"                      |
|  Contains: Seemingly sensitive but expendable     |
|            files (financial docs, personal photos)|
|                                                   |
|  +--------------------------------------------+  |
|  |          Hidden Volume (Secret)             |  |
|  |  Password: "real_password"                  |  |
|  |  Contains: Actually sensitive files         |  |
|  |  (encryption keys, stolen data,            |  |
|  |   communications logs)                      |  |
|  +--------------------------------------------+  |
|                                                   |
+--------------------------------------------------+
```

#### How Plausible Deniability Works

1. **Outer Volume:** Created first, encrypted with one password. Contains believable "sensitive" data.
2. **Hidden Volume:** Created within the free space of the outer volume, encrypted with a different password.
3. **No Evidence:** The hidden volume is indistinguishable from random data filling the outer volume's free space. There is no header, signature, or metadata that reveals its existence.
4. **Password Determines Volume:** Entering the outer password mounts the outer volume. Entering the hidden password mounts the hidden volume. Without the hidden password, its existence cannot be proven.

#### Hidden Operating System

VeraCrypt can create an entire hidden operating system:

- The computer appears to have a single encrypted OS (decoy).
- A hidden OS exists within the same partition, accessible only with a different pre-boot authentication password.
- The decoy OS should contain normal activity to appear legitimate.
- The hidden OS contains the actual sensitive work.

import email
import sys

    msg = email.message_from_file(f)

received = msg.get_all('Received', [])
for i, r in enumerate(reversed(received)):

### VeraCrypt Operations

```bash
# Create an encrypted container (command line)
veracrypt --text --create /path/to/container.vc \
  --size=1G --encryption=AES-Twofish-Serpent \
  --hash=SHA-512 --filesystem=ext4 \
  --password="outer_volume_password" --pim=0 --keyfiles=""

# Create hidden volume within the container
veracrypt --text --create /path/to/container.vc \
  --size=500M --encryption=AES \
  --hash=SHA-512 --filesystem=ext4 \
  --password="hidden_volume_password" --pim=0 --keyfiles="" \
  --volume-type=hidden

# Mount outer volume
veracrypt --text /path/to/container.vc /mnt/veracrypt \
  --password="outer_volume_password" --pim=0 --keyfiles=""

# Mount hidden volume
veracrypt --text /path/to/container.vc /mnt/veracrypt \
  --password="hidden_volume_password" --pim=0 --keyfiles=""

# Dismount
veracrypt --text --dismount /mnt/veracrypt
```

### LUKS Encryption

```bash
# Create LUKS encrypted partition
sudo cryptsetup luksFormat /dev/sdb1 --cipher aes-xts-plain64 --key-size 512 --hash sha512

# Open/unlock the LUKS partition
sudo cryptsetup luksOpen /dev/sdb1 encrypted_vol

# Create filesystem
sudo mkfs.ext4 /dev/mapper/encrypted_vol

# Mount
sudo mount /dev/mapper/encrypted_vol /mnt/encrypted

# Add additional key slot
sudo cryptsetup luksAddKey /dev/sdb1

# Close/lock the partition
sudo umount /mnt/encrypted
sudo cryptsetup luksClose encrypted_vol

# Backup LUKS header (critical for recovery)
sudo cryptsetup luksHeaderBackup /dev/sdb1 --header-backup-file header_backup.img
```

## Real-World Parallels

### Email Forensics in Investigations

- **Sony Pictures Hack (2014):** Email header analysis and forensic examination of the leaked emails helped attribute the attack to North Korea's Lazarus Group. The headers revealed IP addresses and routing information linked to known North Korean infrastructure.
- **Hillary Clinton Email Investigation (2016):** FBI analysis of email headers and server logs was central to the investigation into the use of a private email server for classified communications.
- **Business Email Compromise (BEC):** The FBI reports that BEC attacks have caused over $50 billion in losses globally. Email header analysis is the primary forensic technique for identifying spoofed emails and tracing the actual sender.

### ICS/SCADA Attacks

- **Stuxnet (2010):** The most famous ICS attack, targeting Iranian nuclear centrifuges. Used multiple zero-day exploits and deep knowledge of Siemens S7-300 PLCs to cause centrifuges to spin out of control while reporting normal operations to the HMI.
- **Ukraine Power Grid Attack (2015):** Russian hackers (Sandworm/APT28) attacked Ukrainian power distribution companies, using BlackEnergy malware to gain access to SCADA systems and remotely opening circuit breakers, causing power outages for 230,000 customers.
- **TRITON/TRISIS (2017):** Malware specifically targeting Schneider Electric's Triconex Safety Instrumented System (SIS) at a petrochemical plant in Saudi Arabia. This was particularly alarming because SIS systems are the last line of defense against catastrophic industrial accidents.
- **Colonial Pipeline (2021):** Ransomware attack that disrupted fuel supply across the US East Coast. While the OT systems were not directly compromised, the incident demonstrated the cascading effects of cyberattacks on critical infrastructure.

### Encrypted File Systems and Plausible Deniability

- **TrueCrypt and the FBI:** TrueCrypt's hidden volume feature famously frustrated law enforcement investigations. In several cases, suspects have claimed to provide the "correct" password (to the outer volume) while the hidden volume containing actual evidence remained undetectable.
- **Silk Road Investigation:** Ross Ulbricht's laptop was seized while open and unlocked in a public library, specifically to avoid the challenge of breaking full-disk encryption. This operational tactic by the FBI acknowledged the strength of modern encryption.
- **LUKS Nuke Patch:** A modified version of LUKS allows for a "nuke password" that, when entered, destroys the encryption key, making the data permanently inaccessible. This was developed for journalists and activists in hostile environments.
